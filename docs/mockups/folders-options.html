Folders — placement options
Spaces have linked folders feeding them. Where should the list live? Four candidates, same data in each. Sessions sit on top in every variant — that's not the question. The question is where the Folders list sits relative to Artefacts and how the primary folder reads.
Option A
Folders as own section, below Artefacts
Honest information hierarchy: Sessions → Artefacts → Memories → Folders. Plumbing at the bottom. Full row treatment per folder (path, attach time, detach button).

Recommendation
Ship C. Chip strip is the right primitive — it answers "what's feeding this?" in a single glance, keeps "zero folders" prominent, and stays out of the way otherwise. The popover gives power users the audit detail they'd lose vs A.

Primary: namesake match (oyster-os for the oyster space) → fallback to oldest added_at. Marker is filled chip + ★. No behaviour change yet — purely visual; gives #270/#271 something to anchor when "primary" gains semantic meaning.

D is tempting but adds gesture ambiguity (click = filter? or click = popover?). Worth filing as a follow-up if folder-filter ever becomes a real ask. Start with C; promote to D later if users explicitly want filter-by-folder.
