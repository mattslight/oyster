Try it:
  • Click any tile → Sessions and Artefacts scope to it · click the selected tile again → back to All
  • Hover a tile → ⋯ appears top-right · click → Reveal in Finder, Re-scan, Detach
  • oyster (scratchpad) tile shows the userland-managed folder for this space — always present, can't be detached (no Detach in its ⋯ menu)
  • Click + Attach folder tile → mock alert (real impl: inline path input)
What this brings together:
  • Same primitive at every nav level. Home → space cards. Space → project tiles. Scope is always a tile click.
  • "All" tile replaces the filter bar. The row itself shows the active scope — exactly one tile highlighted, always. No floating banner.
  • Scratchpad is its own thing. The userland folder where Oyster stores natively-created artefacts becomes a first-class tile, distinct from linked repos. Reads as "your space's own stuff."
  • One attach path, one detach path. + tile = attach. ⋯ menu = detach. The scratchpad tile's menu hides Detach because it can't be detached.
  • Pip-level counts on each tile answer "what's live in oyster-os right now" at a glance.

Detach folder?

Its artefacts will be hidden. Reattach the same path to restore them.
