What this gives us, and what it doesn't:
  • One pattern. Click anything that looks tile-shaped or chip-shaped → slide-panel inspector. Sessions, artefacts, memories, folders all behave the same.
  • Chip is unambiguous. Click = inspect. No filter, no popover, no disclosure. The source-origin filter (mine / agents / linked) lives where it always did, on the section heading.
  • Detach + Re-scan + Reveal are inspector actions, not row actions. Less click-target competition on the chip.
  • Empty state still replaces the strip with the red "no folders attached" warning when zero — that detection job is preserved.
  • Primary marker is filled chip + ★ on the strip; primary badge in the inspector title. Visual only for now; #270/#271 can later promote it to "default folder for new artefacts."
What we drop from earlier sketches:
  • The "Show folder details" disclosure (kitchen-sink — adds state without adding clarity).
  • Click-chip-to-filter (overloads click; the source-origin filter already covers most filtering needs).
  • Hover popover for path (inspector handles it; tooltip stays as a minimal hint).
