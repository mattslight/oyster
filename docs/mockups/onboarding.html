Design notes

  • Header copy is user-centric, not protocol-centric. No "MCP server" in the headline — Merlin doesn't need to know the plumbing.
  • Direction of action is explicit: "Connect Oyster to your agent", not "Connect your agent". Makes it clear which thing is being plugged into what.
  • Step 2 has two sub-steps:
    • 2a (optional) — Give Oyster a head start. Drop a folder, instant scan, no agent required. For users who want to see something work immediately.
    • 2b — Ask your agent to set things up. The canonical flow. Copyable prompt that invokes Oyster's MCP tools via the agent connected in step 1. Reinforces the "agent-driven workspace" pitch.
    AND / OR framing — not either-or. User can do both, or pick.
  • Step 3 framing: optional in the step title, trust-first copy above the CTA, green-tinted note reinforces privacy.
  • Completion labels (e.g. "· 3 spaces", "· Claude Code") give the user feedback their action registered.
  • Surface integration: sits as a centered card on the home space, above the builtin tiles (connect-your-ai, quick-start, etc.). Builtins stay below as reference material. Same mount point as today's OnboardingBanner.
  • Dismissal: checklist collapses to a tiny "Show setup" row, desktop returns to normal.
  • Completion detection (story A7) is polish — a manual confirm button is the fallback if SSE/poll detection is flaky.
  • Alternative patterns evaluated: horizontal stepper (more compact, forces linearity), swipe-card pagination (hides arc), docked side panel (steals real estate), chat-style wizard (too long). Accordion wins on "shows the arc + dismissable + fits current mount".
